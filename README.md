# SMO Simulator

Графическое приложение на Python для моделирования системы массового обслуживания (СМО). Программа выполняет дискретно-событийную симуляцию потока заявок, очереди и обслуживающих каналов, сохраняет результаты и строит расчёты/визуализации по сохранённым данным.

## Установка

1. Установите Python 3.10 или новее.
2. Перейдите в корневой каталог проекта:

```bash
cd smo_simulator
```

3.  Установите зависимости:

```bash
pip install -r requirements.txt
```

5. Запустите приложение:

```bash
python main.py
```

## Функционал программы

- Настройка параметров СМО:
  - интенсивность входящего потока заявок `lambda`, заявок в час;
  - размер очереди;
  - длительность моделирования в часах;
  - типы обслуживающих каналов, их количество и интенсивность обслуживания `mu`;
  - политика выбора свободного канала: `round_robin` или `fastest`;
  - дополнительные параметры симуляции: `seed`, первая заявка в момент `t=0`, доработка заявок после конца интервала, ограничение количества заявок.
- Запуск дискретно-событийной симуляции.
- Просмотр краткой сводки: сколько заявок поступило, обслужено и получило отказ.
- Сохранение результатов симуляции в JSON.
- Расчёт показателей по сохранённому JSON:
  - вероятность обслуживания;
  - пропускная способность;
  - вероятность отказа;
  - вероятности занятости каналов;
  - среднее число занятых каналов;
  - вероятности простоя;
  - средняя длина очереди;
  - вероятности состояний очереди;
  - среднее время ожидания, обслуживания и пребывания в системе;
  - среднее число заявок в системе.
- Поддержка warm-up периода: без прогрева, пропуск первых N часов или первых N заявок.
- Сохранение отчёта расчётов в TXT и JSON.
- Построение визуализаций:
  - HTML-временная диаграмма по одному результату симуляции;
  - HTML-графики рассчитанных параметров по нескольким экспериментам;
  - экспорт временной диаграммы в PNG;
  - экспорт данных временной диаграммы в XLSX;
  - массовый экспорт параметров и количества заявок в XLSX.

## Как пользоваться

### 1. Настроить модель

Откройте вкладку **Симуляция** и нажмите **Настройка параметров**.

Заполните основные поля:

- **Интенсивность событий (lambda)** - среднее количество поступающих заявок в час.
- **Размер очереди** - максимальное количество заявок, которые могут ожидать обслуживания.
- **Продолжительность** - время моделирования в часах.
- **Каналы** - добавьте один или несколько типов каналов, задайте для каждого `mu` и количество.

При необходимости настройте дополнительные параметры:

- **Политика свободных серверов**:
  - `round_robin` - равномерное распределение по свободным каналам;
  - `fastest` - выбор канала с наибольшей интенсивностью обслуживания.
- **Drain** - продолжать обслуживание уже поступивших заявок после окончания интервала моделирования.
- **Seed** - фиксирует генератор случайных чисел для воспроизводимости результата.
- **Первая заявка в t=0** - создаёт первую заявку в начальный момент времени.
- **Max arrivals** - ограничивает максимальное количество поступивших заявок.

Нажмите **Сохранить**. Основной конфиг будет записан в `config.json`.

### 2. Запустить симуляцию

На вкладке **Симуляция** нажмите **Запустить симуляцию**.

После выполнения программа покажет краткую сводку:

- количество поступивших заявок;
- количество обслуженных заявок;
- количество отказов.

Чтобы сохранить подробный результат, нажмите **Сохранить результаты** и выберите путь для JSON-файла.

### 3. Выполнить расчёты

Перейдите на вкладку **Расчёты**.

1. Нажмите **Выбрать JSON результатов симуляции** и укажите файл, сохранённый после симуляции.
2. Выберите нужные показатели галочками.
3. При необходимости задайте warm-up период.
4. Нажмите **Запомнить выбор**, если хотите сохранить набор показателей в `calc_config.json`.
5. Нажмите **Рассчитать**.

После расчёта можно сохранить отчёт:

- **Сохранить в TXT** - текстовый отчёт;
- **Сохранить в JSON** - исходный JSON, дополненный блоком `calculations`.

### 4. Построить визуализацию

Перейдите на вкладку **Визуализация**.

Доступные действия:

- **Построить временную диаграмму** - выбрать JSON симуляции и сохранить HTML-страницу с временной диаграммой.
- **Построить графики параметров** - выбрать несколько JSON-файлов с блоком `calculations` и сохранить HTML-страницу с графиками.
- **Экспорт данных в XLSX** - сохранить табличные данные временной диаграммы.
- **Экспорт диаграммы в PNG** - сохранить временную диаграмму как изображение.
- **Массовая выгрузка параметров в XLSX** - собрать рассчитанные показатели по нескольким экспериментам.
- **Массовая выгрузка количества заявок в XLSX** - собрать количество обслуженных и отказных заявок по нескольким экспериментам.

## Структура проекта

- `main.py` - точка входа в приложение.
- `ui/` - окна и вкладки графического интерфейса.
- `simulation/` - ядро дискретно-событийной симуляции и модели данных.
- `visualization/` - генерация HTML, PNG и XLSX-экспортов.
- `config.json` - текущие параметры модели.
- `calc_config.json` - сохранённый набор показателей для расчётов.
- `results/` - каталог для результатов экспериментов, если используется в рабочем процессе.
